## Confirmed @expo/ui API (reusable for Phase 2)
- iOS: `@expo/ui/swift-ui` + modifiers from `@expo/ui/swift-ui/modifiers` (`buttonStyle`, `pickerStyle`, `tint`, `disabled`, `tag`, accessibility). `<Host matchContents>` required to bridge SwiftUI into RN layout. `Picker` selection is value-based via `tag()`. `Alert` slot-based, needs Host; `BottomSheet` self-contained (presentationDetents).
- Android: `@expo/ui/jetpack-compose` (`Button`/`OutlinedButton`/`TextButton`, `ExposedDropdownMenuBox`+`OutlinedTextField`, `ModalBottomSheet`, `AlertDialog`, `SnackbarHost`); `<Host>` required; `OutlinedTextField` value is an `ObservableState` (needs `useNativeState`). ⚠️ RN scroll views may not scroll inside the Android `ModalBottomSheet` (expo/expo#46379).
- Colors flow from `theme/native.ts` only. Each platform-split primitive needs a `jest.moduleNameMapper` entry → bare `.tsx`.

## DEVICE GATE (user) — required before Phase 2
Build a custom Dev Client (not Expo Go) for **both** iOS and Android, run, and verify + screenshot. Per-primitive checks (aggregated from the spikes):
- **burnt (iOS toast):** does it BUILD and show a native HUD? — the single highest-risk unknown. If it fails, fall back to `react-native-toast-message`.
- **ZButton:** Host width (may need a `fullWidth`/`alignSelf:'stretch'` wrapper); RN icon node inside SwiftUI/Compose (may need ZSymbol instead); loading spinner sizing; secondary `tint` on bordered; danger=red.
- **ZSelect:** menu (not wheel) renders; selected label vs placeholder display (iOS `label`-as-placeholder behavior); Android read-only field does not summon the keyboard; `useNativeState` one-frame lag on programmatic value change.
- **Confirm cluster:** Alert destructive=red; iOS Alert has no native button-disable (relies on caller `isPending`); keyboard + textarea inside the iOS sheet (booking-cancel); sheet detent sizing (medium vs fitToContents); Android sheet height (`skipPartiallyExpanded`).
- **toast (Android):** SnackbarHost ref timing on first toast; per-tone color (API shows one chrome for all tones).
- **profile screen:** section-header icons render as native symbols; reports-row Touchable ripple/opacity; save/sign-out button icons visible on their backgrounds.
