# PR #15 Review Fixes

## Context

Code review of PR #15 (mobile token auth + app skeleton + client auth + videos + upload flow) surfaced two medium and several low findings. This change addresses all of them on the PR branch.

## Decisions

- **Refresh resilience (medium):** `refreshTokens` in the mobile client now returns a tri-state outcome (`rotated` / `rejected` / `unavailable`). Only a deliberate WorkOS rejection (400/401) clears the stored pair and signs the user out; 5xx, network failures, and malformed bodies keep the tokens and surface the original 401.
- **Rate limiting (medium):** new per-IP token-bucket limiter (`internal/auth/ratelimit.go`, `golang.org/x/time/rate`, 10 burst / 10 per minute, 10-minute idle pruning) on `POST /auth/token` and `POST /auth/token/refresh`. Client IP comes from the first `X-Forwarded-For` entry, falling back to `RemoteAddr`. No IPs in log lines. 429s documented in `docs/openapi.yaml`.
- **Capability URL hygiene (low):** the upload store drops a file's Mux upload URL once the file reaches `done`, so the AsyncStorage-persisted queue only retains URLs that may still be needed for retries.
- **Store scoping (low):** the upload `processing` re-entrancy set moved from module scope into the store closure; `retryFile`'s whole-job retry semantics are now documented.
- **SecureStore ceiling (low):** dev-only warning when an access token exceeds 2048 bytes.
- **Log hygiene (low):** `auth_token_refresh_succeeded` omits `user_id` when claim parsing fails instead of logging an empty string.
- **Color tokens (low):** new `mobile/src/theme/colors.ts` mirrors the NativeWind CSS variables for icon props/navigator options; all hardcoded hex values in screens/components replaced.
